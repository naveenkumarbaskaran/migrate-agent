# migrate-agent-ai

An AI-powered CLI tool that generates safe **UP** and **DOWN** database migration scripts between two SQL schema versions using [Claude](https://www.anthropic.com/claude) (claude-sonnet-4-6).

## Features

- Diffs two `CREATE TABLE` schemas (added/dropped tables, added/dropped/altered columns)
- Calls Claude with three tools: `read_file`, `write_file`, `validate_sql`
- Detects destructive operations (`DROP TABLE`, `DROP COLUMN`, `TRUNCATE`, type changes) and adds warning comments
- Supports **PostgreSQL**, **MySQL**, and **SQLite** dialects
- Rich terminal output with syntax highlighting

## Quick Start

```bash
export ANTHROPIC_API_KEY="sk-ant-..."

# Generate a migration
migrate-agent plan --from current.sql --to target.sql --dialect postgres --output migration.sql

# Just show the schema diff (no AI call)
migrate-agent plan --from current.sql --to target.sql --diff-only

# Standalone diff subcommand
migrate-agent diff --from current.sql --to target.sql
```

## Example

Given **current.sql**:

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) NOT NULL
);
```

And **target.sql**:

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    email VARCHAR(255) NOT NULL UNIQUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INT NOT NULL REFERENCES users(id),
    title TEXT NOT NULL,
    body TEXT
);
```

Running:

```bash
migrate-agent plan --from current.sql --to target.sql
```

Produces a `migration.sql` like:

```sql
-- Generated by migrate-agent-ai
-- Dialect: postgres
-- From: current.sql  To: target.sql

-- === UP MIGRATION ===

ALTER TABLE users
    ALTER COLUMN name TYPE VARCHAR(200),
    ADD COLUMN created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    ADD CONSTRAINT users_email_unique UNIQUE (email);

CREATE TABLE IF NOT EXISTS posts (
    id SERIAL PRIMARY KEY,
    user_id INT NOT NULL REFERENCES users(id),
    title TEXT NOT NULL,
    body TEXT
);

-- === DOWN MIGRATION ===

DROP TABLE IF EXISTS posts;

-- WARNING: Destructive operation -- verify before applying
ALTER TABLE users
    DROP COLUMN created_at,
    DROP CONSTRAINT users_email_unique,
    -- WARNING: type change may lose data
    ALTER COLUMN name TYPE VARCHAR(100);
```

## Python API

```python
from migrate_agent import MigrateAgent, SchemaDiffer

# Use the agent programmatically
agent = MigrateAgent()  # reads ANTHROPIC_API_KEY from env
summary = agent.plan(
    from_schema="current.sql",
    to_schema="target.sql",
    dialect="postgres",
    output="migration.sql",
)
print(summary)

# Use the differ directly (no AI, no API key needed)
differ = SchemaDiffer()
diff = differ.diff(current_sql_text, target_sql_text)
print(diff)  # human-readable summary
```

## Architecture

```
migrate_agent/
├── __init__.py      # package exports
├── agent.py         # MigrateAgent + tool implementations + agentic loop
├── differ.py        # SchemaDiffer -- pure-Python SQL parser & diff
└── cli.py           # Click CLI with Rich output
```

### Tools available to the agent

| Tool | Description |
|---|---|
| `read_file(path)` | Read a SQL schema file from disk |
| `write_file(path, content)` | Write the finished migration to disk |
| `validate_sql(sql, dialect)` | Detect destructive ops, unbalanced parens, empty scripts |

### Agentic loop

1. Claude receives the diff summary and instructions.
2. Claude calls `read_file` for both schema files.
3. Claude analyses the changes and drafts the migration SQL.
4. Claude calls `validate_sql` -- warnings are added as SQL comments.
5. Claude calls `write_file` to save the output.
6. Claude returns a text summary; the CLI displays the file with syntax highlighting.
